**프로젝트명:** Three Card Survival
**장르:** 덱빌딩 생존 카드게임
**플랫폼:** PC (Unity 프로토타입)

---

## 1. 핵심 컨셉

매 턴 3장의 카드 중 1장을 선택해 자원을 관리하며 생존하는 덱빌딩 게임.
선택하지 않은 2장은 다음 턴으로 이월된다. 덱이 소진되어도 게임은 이월 카드만으로 계속된다.
하루의 끝 카드를 선택했을 때 탈출 시도가 가능하며, 탈출 성공 시 게임 클리어.

## 5. 핵심 시스템

### 덱 관리 (DeckManager)
- 매 턴 3장 공개: carriedOver(이월) + 덱 상단에서 보충
- 카드 선택 시: 선택 → usedCards, 나머지 2장 → 다음 턴 carriedOver
- 덱 소진 후에도 이월 카드만으로 계속 진행

### 카드 선택 가능 조건 (CardSelector)
- costResource가 지정되어 있으면: 해당 자원 >= costAmount
- requiredStat이 지정되어 있으면: 해당 스탯 >= requiredStatAmount
- 3장 모두 선택 불가 → 게임오버

### 하루의 끝 선택지 (EffectResolver)
1. 하루를 끝내기 → DayEndRoutine 실행
2. 계속 진행하기 → 하루의 끝 카드를 덱에 다시 섞고 턴 계속
3. 탈출 시도 → EscapeSystem.TryEscape() 호출

### 하루 종료 루틴 (DayEndRoutine)
1. 카드 보상: 새 카드 3장 삼중택일 → 선택 카드 덱에 추가
2. 카드 합성: 덱에서 2장 선택 → 1장으로 합성
3. 카드 강화: 덱에서 1장 선택 → 효과량 +1 (임시)
4. 시련 추가: 랜덤 시련 카드 1장 덱에 추가
5. 마일스톤 체크: 5일차 → SOS 카드 덱에 추가
6. day + 1 → 다음 날 시작

### 탈출 시스템 (EscapeSystem)
- 성공 조건: Random.value < escapeChance
- 탈출 확률 공식 (임시):
  - escapeChance += (STR + AGI + INT) * 0.01
  - escapeChance += (Food + Water) * 0.005
- 실패 시: 이후 모든 카드 costAmount +1 (리스크 누적)

---

## 6. 게임 흐름 (GameManager 상태 머신)

```
Start → 초기 덱 생성 → PlayerTurn
PlayerTurn → 카드 선택 → 효과 처리 → 게임오버 체크 → PlayerTurn 반복
              └─ 하루의 끝 선택 시 → DayEndRoutine / EscapeSystem
DayEndRoutine 완료 → PlayerTurn
```

**상태 목록:** PlayerTurn / DayEndRoutine / GameOver / GameClear
