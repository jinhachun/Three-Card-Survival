**프로젝트명:** Three Card Survival
**장르:** 덱빌딩 생존 카드게임
**플랫폼:** PC (Unity 프로토타입)

---

## 1. 핵심 컨셉

매 턴 3장의 카드 중 1장을 선택해 자원을 관리하며 생존하는 덱빌딩 게임.
선택하지 않은 2장은 다음 턴으로 이월된다. 덱이 소진되어도 게임은 이월 카드만으로 계속된다.
하루의 끝 카드를 선택했을 때 탈출 시도가 가능하며, 탈출 성공 시 게임 클리어.

## 5. 핵심 시스템

### 덱 관리 (DeckManager)
- 매 턴 3장 공개: carriedOver(이월) + 덱 상단에서 보충
- 카드 선택 시: 선택 → usedCards, 나머지 2장 → 다음 턴 carriedOver
- 같은 이름 카드 자동 중첩 표시 (×N 배지)
- 덱 소진 후에도 이월 카드만으로 계속 진행

### 시련 카드 패널티
- 이월 중인 시련 카드가 있을 때 카드를 선택할 때마다 HP -1 (중첩 수만큼)
- 시련 카드를 직접 선택(비용 지불)하면 즉시 해소

### 카드 선택 가능 조건
- 자원 비용 충족 여부
- 스탯 조건 충족 여부 (예: Strength ≥ 2)
- 3장 모두 선택 불가 → 게임오버

### 카드 보상 풀 필터링
- minDay 조건: 해당 날짜 미만이면 풀에서 제외
- requiredBuilding 조건: 해당 건물이 완성되어야 풀에 등장
- 이미 완성된 건물의 Build 카드는 풀에서 제외

### 건물 시스템
- Buildings.csv → CsvBuildingImporter로 BuildingData 에셋 + "Build X" 카드 자동 생성
- 건물 완성 시: 스탯 획득 또는 패시브 효과 활성화
- 패시브 건물(Well/Campfire/Infirmary): 매 카드 선택마다 자원 +1
- 스탯 건물(Gym/Training Ground/Library): 완성 시 스탯 +1, 관련 카드 보상 풀 해금
- 완성 연출: BuildingCompletePopup 팝업 (DOTween)
- 패시브 발동 연출: PassiveNotifier 플로팅 텍스트

### 하루의 끝 선택지
1. End Day → DayEndRoutine 실행
2. Continue → 하루의 끝 카드를 덱에 다시 섞고 턴 계속
3. Try Escape → EscapeSystem.TryEscape() 호출

### 하루 종료 루틴 (DayEndRoutine)
1. 카드 보상: 보상 풀에서 3장 제시 → 1장 선택 → 덱에 추가
2. 시련 추가: 랜덤 시련 카드 1장 덱에 추가
3. 마일스톤: 5일차 → SOS 카드 덱에 추가
4. HP 전량 회복, day +1, 덱 셔플

### 탈출 시스템 (EscapeSystem)
- 성공 조건: Random.value < escapeChance
- escapeChance는 SOS 카드 사용마다 +10%
- 실패 시: 이후 모든 카드 비용 +1 누적 (costPenalty)

## 6. 게임 흐름

```
Start → 초기 덱 생성 → PlayerTurn
PlayerTurn → 카드 선택 → 효과 처리 → 게임오버 체크 → PlayerTurn 반복
              └─ 하루의 끝 선택 시 → DayEndRoutine / EscapeSystem
DayEndRoutine 완료 → PlayerTurn
```

**상태 목록:** PlayerTurn / DayEndChoice / DayEndRoutine / GameOver / GameClear

---

## 7. 데이터 관리

- **Cards.csv** → CsvCardImporter → CardData 에셋 + CardRegistry
- **Buildings.csv** → CsvBuildingImporter → BuildingData 에셋 + "Build X" CardData + BuildingRegistry
- 전체 임포트: `ThreeCardSurvival/전체 임포트 (카드 + 건물)` 메뉴
